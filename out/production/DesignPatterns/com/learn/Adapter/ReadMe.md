### 适配器模式
        将一个类的接口转换成客户希望的另一个接口
        
        Adapter模式使得原本由于接口不兼容不能一起工作的那些类可以一起工作。
        
        其别名为包装器（Wrapper）
        
        适配器模式既可以作为类结构型模式，也可以作为对象结构型模式
        
#### 1、类适配器
        使用多重继承对一个接口与另一个接口进行匹配
                
#### 2、对象适配器
        对象匹配依赖于对象组合
        
#### 3、优势
        适配器类，可使任何两个无直接联系的类在一起运行
        
        目标（target)角色可通过被适配者（Adaptee）具体实现
        
        适配器类，大大提高了类的复用度与灵活性
