### 原型模式
        是指用原型实例制定创建对象的种类，并且通过拷贝这些原型创建新的对象
        
        “用原型实例制定创建对象的种类”可以理解为：新建一个对象，运用一个原型指定对象的种类。
        
        “通过拷贝这些原型创建新的对象”可以理解为：通过现有对象的复制实现新对象的创建。
        
#### 优点
        1、允许动态的增加或减少产品类。
        
        2、提供简化的创建结构。
        
        3、具有给一个应用软件动态加载新功能的能力
        
        4、产品类不需要非得有任何事先确定的等级结构，因为原型模式适用于任何的等级结构
